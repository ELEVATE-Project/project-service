# 🚀 Project-Service Release 3.4.1 [![Latest](https://img.shields.io/badge/Latest-ffffff00?style=flat&labelColor=ffffff00&color=green)](#)

## ✨ Features & Improvements

-   **[1767] Optimize Start Improvement (`userprojects/details`) API**
    Improved the performance and response time of the Start Improvement API.

-   **[1766] Internal Caching and Optimization of Program Listing API – Project Service**
    Added internal caching and optimizations to improve efficiency and reduce response latency for program listing APIs.

-   **[1764] Internal Caching and Optimization of Project Listing API**
    Enhanced project listing APIs with internal caching and performance optimizations.

-   **[1740] Enable Tenant Admin to Create Resources in Project Service**
    Added support for tenant administrators to create and manage resources within the Project Service.

-   **[1739] Refine Multi-user Access to Project for Tenant-based Architecture**
    Refined multi-user project access flow to better align with the tenant-based architecture and access model.

---

## 🐞 Bug Fixes

-   **[4564] Program Manager Mapping Issue for Multi-org and Multi-state Programs**
    Fixed an issue where Program Managers were not mapped correctly for programs associated with multiple organizations and states.

-   **[3204] Missing Backend Validation for Project Creation**
    Fixed an issue where projects could be created with empty title and objective fields by adding proper backend validations.
